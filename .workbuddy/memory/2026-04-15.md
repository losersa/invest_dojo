#### 图表修复 (KLineChart.tsx)

## KLineChart 分钟K线排序崩溃 Bug 修复
- 根因：`markers.sort()` 用 `localeCompare()` 但 5min 数据的 time 是 `number` 类型（Unix 秒数），不是 string
- 同时修复：新闻标记在分钟模式下无法匹配（`"1577947500"` vs `"2020-01-20"`）
- 修复方案：
  1. `toChartTime()` 同时接受 `string | number` 输入
  2. 新增 `compareTime()` 函数兼容 number/string 排序
  3. 新增 `isMinuteData()` 判断当前是否分钟级数据
  4. 新增 `unixToDate()` 从 Unix 秒数提取日期
  5. 新闻匹配逻辑分分钟/日K两个分支处理
- 验证：两个场景页面 HTTP 200 + 零 lint 错误

## React "Cannot update during render" Bug 修复
- 报错：`Cannot update a component (SimulationPlayground) while rendering a different component`
- 根因 3 个：
  1. `use(paramsPromise)` 在客户端组件中使用，与 Next.js 15 SSR 水合冲突
  2. render 函数体中两个 IIFE (chartKlines / minuteProgressInfo) 在渲染阶段间接触发 state 更新链
  3. setInterval 回调中在 `setMinutePlayIndex` 的 setState 回调里直接调用 `advanceDay()`（Zustand setState），违反 React 规则
- 修复：
  1. `page.tsx` 改为 async Server Component，`await params` 后把 string 传给客户端组件
  2. 两个 IIFE 改为 `useMemo`
  3. setInterval 中 advanceDay 用 `setTimeout(() => advanceDay(), 0)` 推到下一个微任务
  4. currentDate 重置用 `useRef` 追踪前值，避免初始化竞争
  5. `minuteKlines` → `minuteKlinesMap` 变量重命名避免混淆
  6. `latestRef` 合并所有需要在 interval 中读取的状态

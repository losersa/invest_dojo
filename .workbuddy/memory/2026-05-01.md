### 架构关键决策
**选 A 方案**：把 fundamentals JSONB 展平为 DSL 内置字段（`eps_ttm/roe/yoy_ni` 等），
而不是 `fundamental("roeAvg", "profit")` 函数形式。
YAML 可读性 >> 灵活性，通用性靠函数在需要时再加。

**防未来函数**：
- panel_loader 把 `announce_date + 1 天`才让财报对 DSL 可见
- pivot 后 ffill，再合并 kline index 再 reindex（避免 reindex(method='ffill') 的对齐 bug）

### 踩过的坑
1. **panel_loader reindex(method='ffill') bug**：
   三只股票公告日不同 → pivot 后稀疏 → `reindex(method='ffill')` 只能从最近已知位置 ffill，
   但"最近位置"对部分 symbol 也是 NaN，结果每列只有自己公告日那几行有数据。
   解：**先对每列独立 ffill**，再合并 kline_index 再 reindex → 每天每 symbol 都填满

2. **compute/validate 漏传 needed_fields**：panel_loader 不知道要加载基本面，统统 NaN → 503

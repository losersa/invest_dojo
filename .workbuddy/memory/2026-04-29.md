## T-1.04 5m K 补采 + 场景重构 · 已完成

### 关键决策

  - 股票池：000002 万科A / 601318 中国平安 / 600519 茅台
- `ai_boom_2023`（2023-02 ~ 2023-06）替代 bull_2014，代表"疯牛题材"
  - 股票池：300308 中际旭创 / 600845 宝信软件 / 300033 同花顺

### 采集工具
- `scripts/seed_scenario_data.py` —— 从 scenarios 表读取定义自动采集
- 核心改进：
  - `signal.SIGALRM` 超时保护（避开 BaoStock socket 死循环）
  - 3 次重试，失败自动重 login
  - 断点续传（查 COUNT(*) 跳过已有）

### 数据库改动（重要）
**修复了 klines_all 唯一约束**：
- 旧：`UNIQUE (symbol, timeframe, dt)`  ← 不允许场景数据和全市场数据并存
- 新：`UNIQUE (scenario_id, symbol, timeframe, dt)` + partial index on NULL scenario_id
- 原因：旧约束导致 `ai_boom_2023` 采 600845 日K 撞上全市场已有的 600845 数据

### 踩坑
1. **BaoStock recv 死循环**：`baostock/util/socketutil.py:67` `while True: socket.recv()` 没 timeout。用 `signal.SIGALRM` 在 Python 层强制打断
2. **BaoStock 5m 不支持 pctChg 字段**：传了会返回 `10004012 5分钟线指标参数传入错误:pctChg`。日K 支持，5m 不支持
3. **5m 返回 9 字段含 date+time**：要求的是 `date,time,open,...`，返回时 time 在第 2 位
4. **之前 covid_2020 和 new_energy_2020 能共存**：是因为它们 dt 用 `T15:00:00+08:00`，全市场数据用 `T00:00:00+08:00`（UTC 偏移不同 = 时间戳不同）。这是偶然，改约束才是正道

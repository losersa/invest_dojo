## 修复模拟交易自动播放 4 个问题

### 1. 分钟播放后期变"按天"推进（核心 bug）
- **根因**：`setMinutePlayIndex` 回调中 `next >= clipped.length` 时 `return 0`，随后 `useEffect [currentDate]` 又把 index 重置为 `clipped.length`，导致下一 tick 立即触发 `advanceDay`，表现为"按天跳"。
- **修复**：进入新一天时**不再重置** `minutePlayIndex`；达到当天末尾时 `return currentLen`（而非 0），随着 `advanceDay` 推进，clipped.length 扩大，index 继续向前。

### 2. 播放到最后应停留、不循环
- **修复**：定时器内判断 `if (s.isLastDay) { setPlayMode("stopped"); return currentLen; }`，最后一天的最后一根 K 线播放完毕后自动暂停，停在最后价格。

### 3. 速度按钮消失 / 某些模块无控制栏
- **根因**：`{has5min && !isLastDay && (...)}` 限制太严格——末日时整栏消失；无 5min 数据的场景（如 mock 场景）完全没有控制栏。
- **修复**：控制栏**始终显示**。5m/1h 按钮在无分钟数据时禁用（opacity 0.3 + title 提示）；末日时播放按钮禁用并显示"🏁 场景结束"。

### 4. K 线图随数据增多被压缩
- **根因**：`chart.timeScale().fitContent()` 每次按全部 K 线自适应，数据越多每根越细。
- **修复**：`KLineChart` 新增 `visibleBars` 属性，用 `setVisibleLogicalRange({ from: len - visibleBars, to: len })` 实现**固定时间窗口**。窗口大小按周期自适应：
  - 5m/15m: 120 根（≈ 10 小时）
  - 1h/4h: 80 根
  - 1d: 60 根（≈ 3 个月）
  - 1w: 52 根（1 年）
  - 1M: 24 根（2 年）
  主图和副图（VOL/MACD/KDJ/RSI）同步应用。

## 修复：步长 5m 但日K图不跟随实时更新

### 问题
步长设为 5m 时，K线图显示日K/周K，图表只在 advanceDay 时更新（整天跳一次），而不是每推进一根 5m 时日K蜡烛实时"生长"。

### 根因
`chartKlines` 的 useMemo 逻辑分两支：
- `timeFrame ∈ [5m,15m,1h,4h]` → 用 5m 数据 + minutePlayIndex 裁剪
- `timeFrame ∈ [1d,1w,1M]` → 直接用 visibleKlines（store 按天裁剪），完全不看 minutePlayIndex

所以 5m 步长 + 日K 视图 = 无实时更新。

### 修复
1. `aggregateMinuteKlines` 扩展支持 `"1d" | "1w" | "1M"` 目标周期（按日期字符串分组，输出 "YYYY-MM-DD" 格式给 Lightweight Charts）
2. `chartKlines` 增加 `isMinuteStep` 判断：当步长为 5m/1h 且有分钟数据时，**所有 timeFrame 都从 5m 数据实时聚合**（包括日K/周K/月K），蜡烛随播放"生长"
3. 步长切为 5m/1h 时自动加载分钟数据缓存
4. 分钟进度信息也在 isMinuteStep 条件下始终显示

### 修改文件
- `apps/web/src/lib/supabase-loader.ts` — aggregateMinuteKlines 支持 1d/1w/1M
- `apps/web/src/app/simulation/[scenarioId]/SimulationPlayground.tsx` — chartKlines 逻辑重构 + 新增 stepSize 切换自动加载 5m 数据

## 修复：K 线到 7-24 就不更新（Supabase 分页 bug）

### 问题
用户切到日K/小时K，K 线数据到 2020-07-24 左右就停住不更新。

### 根因（两层）
1. **Supabase PostgREST 默认单次查询硬限制 1000 行**。`new_energy_2020/300750` 的 5m 数据前 1000 行正好到 2020-07-29 14:20（视觉上就是 7-24 ~ 7-29 区间）。
2. 前端 `loadMinuteKlines` 虽然写了分页循环，但**在循环外构造 query 对象，循环内复用**。supabase-js 的 query builder 是"一次性"的，第一次 `.range()` 调用后就被消费，再次调用条件累加，导致分页失效。

### 修复
- `loadMinuteKlines` 改为**每页重新构造 query**，保证分页正确。

## 闲聊：OrbStack vs Docker Desktop（2026-04-28 晚）

用户问 OrbStack 和 Docker Desktop 的区别以及 Pro 版是否值得买。

### 给出的核心结论
- **技术差异**：OrbStack 基于 macOS 原生虚拟化 + VirtioFS，启动快（秒级 vs 分钟级）、内存占用低（~100MB vs 2-4GB）、挂载性能高；Docker Desktop 跨平台、生态完整、企业合规成熟
- **Pro 值不值**：Pro 版（$8/月）**不解锁任何独占技术功能**，只提供商用授权 + 团队管理 + SSO + 优先支持
- **决策建议**：
  - 个人设备 + 个人项目（如 InvestDojo）→ 免费版够用
  - 公司设备 + 公司项目 → 必须走商用授权（腾讯这种体量属于商用）
  - 腾讯内部通常有统一容器方案（TKE / 蓝盾），公司机器使用前建议先走 IT/合规确认

### 对 InvestDojo 项目的相关性
未来容器化部署 Python 回测引擎时，OrbStack 的 VirtioFS 对 `apps/web/node_modules` 和 Python venv 挂载的性能优势显著，且完全兼容 Docker CLI + Compose，迁移零成本。

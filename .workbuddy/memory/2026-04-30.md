### 核心设计
1. **as_of 注入**（核心红线）：
   - K 线：`dt < as_of`（UTC 时间戳比较）
   - 财报：`announce_date < as_of`（公告日期，不是报告期！）
   - 市场快照：`date < as_of` 严格小于，日期相等直接 403 future_leak
2. **PostgREST filter 组装**：单 dt 条件直接放 filters；多条件合并到 `and=(...)`
3. **时区规范化**：日 K 输出 YYYY-MM-DD（北京日期），分钟 K 保留完整 ISO 8601
4. **分页**：Range header 驱动，单页上限 1000
5. **status 别名映射**：前端 `normal` → DB `active`

### 踩坑
1. **FastAPI 路由顺序**：`/factors/{factor_id}` 会吞掉 `/factors/categories` 的请求，路由必须按静态路径优先注册。方案：把 categories/tags 和 factors 合并到同一 router，写在 {factor_id} 之前。
2. **PostgREST JSONB array 筛选用 `cs.[...]` 不是 `cs.{...}`**：
   - `cs.{hs300}` 是 PostgreSQL native array 语法
   - `cs.["hs300"]` 才是 JSONB array 语法
   - 多值 AND：`cs.["趋势","经典"]`
   - 中文用 `json.dumps(tags, ensure_ascii=False)` 生成
3. **PostgREST 批量 POST `All object keys must match`**：批量对象的 key 集合必须完全一致，缺失字段也要补 null。解决：逐条插入或预先补齐所有字段。
4. **pytest 两个 svc 同名 `common_utils.py` 冲突**：先 import 的占坑。用 `importlib.util.spec_from_file_location` 按绝对路径加载，并在 `sys.modules["common_utils"]` 注入对应版本。

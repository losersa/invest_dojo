# API 通用约定

> 文档状态：**Stable v1.0**
> 最后更新：2026-04-28
> 适用于所有 InvestDojo API

---

## 1. 版本化

- 所有接口 URL 形如 `/api/v{N}/...`
- 当前主版本：**v1**
- 破坏性变更需发布新主版本并在旧版本加 `Deprecation` header

## 2. 认证

所有需要用户身份的接口统一使用 **Bearer Token**（Supabase JWT）：

```http
Authorization: Bearer <supabase_access_token>
```

匿名接口（如首页、公开模型列表）可不带此 header。

## 3. 响应格式

### 3.1 成功 · 单个对象

```json
{
  "data": { ... },
  "meta": {
    "request_id": "req_xxx",
    "timestamp": "ISO 8601"
  }
}
```

### 3.2 成功 · 列表（分页）

```json
{
  "data": [ ... ],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 100,
    "total_pages": 5,
    "has_next": true
  },
  "meta": { ... }
}
```

### 3.3 错误

```json
{
  "error": {
    "code": "BUSINESS_ERROR_CODE",
    "message": "Human-readable message",
    "detail": { ... }
  },
  "meta": {
    "request_id": "req_xxx"
  }
}
```

## 4. HTTP 状态码

| 码 | 语义 |
|----|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 202 | 异步任务已接受 |
| 204 | 成功，无返回体 |
| 400 | 参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 资源冲突 |
| 422 | 业务规则违反 |
| 429 | 速率限制 |
| 500 | 服务器内部错误 |
| 503 | 服务不可用 |

## 5. 幂等性

- `GET` / `DELETE` 天然幂等
- 写操作（`POST` / `PUT` / `PATCH`）如需幂等，传 `Idempotency-Key` header
  - 格式：UUID v4
  - 24h 内相同 key 返回首次结果

## 6. 速率限制

### 6.1 默认额度

| 类型 | 额度 |
|------|------|
| 读接口 | 100 req/min/user |
| 写接口 | 20 req/min/user |
| 计算接口（含因子/训练/回测） | 10 req/min/user |

### 6.2 超限响应

```http
HTTP/1.1 429 Too Many Requests
Retry-After: 30
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1714304400
```

## 7. 分页

- `page` 默认 1，最小 1
- `page_size` 默认 20，最大 100
- 超范围返回空列表，不报错
- **不用** cursor-based 分页（当前数据量下 offset 够用）

## 8. 排序

- 单字段排序：`sort=field_name`
- 降序：`sort=-field_name`
- 多字段（按优先级）：`sort=-updated_at,name`
- 仅允许文档明确列出的字段，否则 400

## 9. 时间格式

- 统一 ISO 8601 with timezone：`2026-04-28T10:31:00Z`
- 日期型：`2026-04-28`
- **不用** Unix timestamp
- 所有时间字段默认 UTC，前端按需本地化

## 10. 数值精度

| 字段类型 | 小数位 | 示例 |
|---------|-------|------|
| 股价 | 4 | `1720.5000` |
| 百分比 | 4（0~1 形式） | `0.0234`（即 2.34%） |
| 成交量 | 整数 | `1234567` |
| 因子值（scalar） | 原始 | 无约束 |

## 11. 字符编码

- Request/Response 均为 UTF-8
- 股票名称、描述等中文字符直接传输，不做 URL encode

## 12. 异步任务

耗时 > 5s 的操作统一走异步模式：

### 12.1 提交
```http
POST /api/v1/xxx
→ 202 Accepted
{
  "data": {
    "job_id": "job_xxx",
    "status": "pending",
    "estimated_seconds": 30,
    "poll_url": "/api/v1/xxx/jobs/job_xxx"
  }
}
```

### 12.2 轮询
```http
GET /api/v1/xxx/jobs/job_xxx
→ 200 OK
{
  "data": {
    "job_id": "job_xxx",
    "status": "running",    // pending/running/completed/failed
    "progress": 0.45,
    "result_url": null,
    "error": null
  }
}
```

### 12.3 取结果
```http
GET /api/v1/xxx/jobs/job_xxx/result
→ 200 OK / 对应资源
```

## 13. WebSocket / SSE

### 13.1 WebSocket 连接
```
wss://api.investdojo.com/ws/v1/sessions/{session_id}/signals
?token=<supabase_access_token>
```

### 13.2 消息格式（JSON）

**服务端推送**：
```json
{
  "type": "signal",        // signal/tick/heartbeat/error
  "payload": { ... },
  "sequence": 1234,
  "timestamp": "ISO 8601"
}
```

**客户端发送**：
```json
{
  "type": "subscribe",
  "payload": { "model_id": "abc" }
}
```

### 13.3 心跳
- 服务端每 30s 发送 `type: heartbeat`
- 客户端 60s 未收到即重连

## 14. CORS

开放：
- `Origin: https://investdojo.com`
- 开发：`Origin: http://localhost:3000`

生产环境禁止 `*`。

## 15. Request ID

每个请求自动分配 `request_id`，写入：
- 响应 `meta.request_id`
- 响应 Header `X-Request-ID`
- 服务端日志

用户报问题时提供 `request_id` 可快速定位。

## 16. 变更通知机制

### 16.1 Deprecation Header
被废弃的接口/字段响应中附带：
```http
Deprecation: true
Sunset: Sat, 28 Oct 2026 00:00:00 GMT
Link: </api/v2/...>; rel="successor-version"
```

### 16.2 文档中标注
```markdown
> ⚠️ **DEPRECATED (v1.5.0)**：此字段将在 v2 移除，请使用 `xxx` 替代。
```

### 16.3 向后兼容承诺
- 非破坏性变更（加字段、加参数、加接口）：随时上线
- 破坏性变更：至少 6 个月过渡期 + 邮件/站内信通知

## 17. 错误码命名规则

格式：`<CATEGORY>_<DETAIL>`

常见类别：
- `AUTH_*` 认证相关
- `PERMISSION_*` 权限
- `VALIDATION_*` 参数校验
- `NOT_FOUND_*` 资源不存在
- `CONFLICT_*` 冲突
- `QUOTA_*` 配额/限流
- `COMPUTE_*` 计算错误
- `INTERNAL_*` 服务器错误

例如：`AUTH_TOKEN_EXPIRED`、`VALIDATION_INVALID_FORMULA`、`QUOTA_RATE_LIMIT_EXCEEDED`。

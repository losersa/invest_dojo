## 1. 数据层构成

```
┌─────────────────────────────────────────────────────────────┐
│                     Supabase (Postgres 15)                   │
│  行情 / 新闻 / 财报 / 用户 / 模型 / 回测 / 会话              │
└─────────────────────────────────────────────────────────────┘
                             │
              ┌──────────────┴──────────────┐
              │                             │
┌─────────────┴────────────────┐ ┌──────────┴──────────────────┐
│    Redis Cluster             │ │   MinIO / S3                │
│  · 热门因子值                  │ │  · 模型文件                  │
│  · 推理结果缓存                │ │  · Notebook 快照             │
│  · 会话实时状态                │ │  · 回测 HTML 报告            │
│  · Pub/Sub 消息               │ │  · 场景封面/图片             │
│  · 速率限制计数                │ │                             │
└──────────────────────────────┘ └─────────────────────────────┘
```

### 1.1 数据量级预估（MVP 规模）

| 数据 | 数量级 | 增长率 |
|------|-------|-------|
| 股票元数据 | ~5,500 只 | 每月 +10 |
| 日 K 线（2014-至今） | ~1,200 万条 | 每日 +5,000 |
| 5m K 线（2014-至今） | ~2.5 亿条 | 每日 +48×股票数 |
| 新闻 | 10 万~100 万 | 每日 +500 |
| 财报 | ~20 万 | 每季度 +5,500 |
| 因子定义 | 200 内置 + 用户自定义 | 用户驱动 |
| 因子值 | ~60 亿 | 日 K × 200 因子 × 3000 股票 |
| 模型 | 官方 5 + 用户 N | 用户驱动 |
| 会话 | 用户 × 月 × 3 次 | 用户驱动 |
| 会话事件 | 会话数 × 100~1000 | 用户驱动 |

### 1.2 分层存储策略

| 数据 | 位置 | 保留 |
|------|------|------|
| 日 K（近 5 年） | Postgres 主表 | 永久 |
| 日 K（> 5 年） | Postgres 分区冷表 / S3 | 永久 |
| 5m K（近 6 个月） | Postgres 主表 | 永久 |
| 5m K（> 6 个月） | S3 Parquet 归档 | 永久 |
| 推理结果 | Redis（2h 过期） | 热数据 |
| 会话实时状态 | Redis（会话结束后落 DB） | 短期 |
| 训练样本 | Postgres | 永久 |
| 模型文件 | MinIO | 永久 |

### 3.2 `klines_all`（现有，保留 + 扩展）

```sql
CREATE TABLE klines_all (
    id              BIGSERIAL PRIMARY KEY,
    scenario_id     TEXT,                       -- 可为空（全市场数据不依赖场景）
    symbol          TEXT NOT NULL,
    timeframe       TEXT NOT NULL,              -- 1m/5m/15m/1h/1d/1w/1M
    dt              TIMESTAMPTZ NOT NULL,
    open            NUMERIC(12,4),
    high            NUMERIC(12,4),
    low             NUMERIC(12,4),
    close           NUMERIC(12,4),
    volume          BIGINT,
    turnover        NUMERIC(20,2),
    pre_close       NUMERIC(12,4),
    change_amount   NUMERIC(12,4),
    change_percent  NUMERIC(8,4),
    adj_factor      NUMERIC(12,6) DEFAULT 1     -- 新增：复权因子
);

-- 必要索引
CREATE UNIQUE INDEX uniq_klines ON klines_all(symbol, timeframe, dt);
CREATE INDEX idx_klines_symbol_timeframe ON klines_all(symbol, timeframe, dt DESC);
CREATE INDEX idx_klines_scenario ON klines_all(scenario_id, symbol, timeframe) WHERE scenario_id IS NOT NULL;
```

**迁移要点**：
- 当前 `scenario_id` 必填，需改为可空（全市场数据不依赖场景）
- 新增 `adj_factor` 列（前/后复权切换）
- 现有数据保留，但未来全 A 股扩展后，大部分数据 `scenario_id = NULL`

### 3.3 `news`（现有，保留 + 扩展）

```sql
CREATE TABLE news (
    id              TEXT PRIMARY KEY,
    scenario_id     TEXT,                  -- 可为空
    published_at    TIMESTAMPTZ NOT NULL,  -- ⭐ 替代原 date（精确到分钟）
    date            DATE NOT NULL,         -- 保留兼容
    title           TEXT NOT NULL,
    content         TEXT,
    source          TEXT,
    category        TEXT,                  -- macro/policy/industry/company/market
    sentiment       TEXT,                  -- positive/neutral/negative
    sentiment_score NUMERIC(5,4),          -- 新增：情绪分 -1~1
    impact_level    INT,                   -- 1-5
    related_symbols JSONB,
    related_industries JSONB,              -- 新增
    tags            JSONB,                 -- 新增
    url             TEXT,                  -- 新增
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_news_published_at ON news(published_at DESC);
CREATE INDEX idx_news_symbol ON news USING GIN (related_symbols);
CREATE INDEX idx_news_fulltext ON news USING GIN (to_tsvector('simple', title || ' ' || coalesce(content,'')));
```

**迁移要点**：
- `date` 保留，但新增 `published_at timestamptz` 做精确时间
- 新增 `sentiment_score`、`related_industries`、`tags`、`url` 列
- 加全文搜索索引

### 4.9 `feature_values` — 因子值矩阵（新增）

```sql
-- 稀疏存储：只存有值的行
-- 预期数据量：~60 亿（200 因子 × 3000 股票 × 10 年）
-- 按日期分区
CREATE TABLE feature_values (
    factor_id       TEXT NOT NULL,
    symbol          TEXT NOT NULL,
    date            DATE NOT NULL,
    value_num       DOUBLE PRECISION,           -- scalar/rank
    value_bool      BOOLEAN,                    -- boolean
    computed_at     TIMESTAMPTZ DEFAULT NOW(),
    PRIMARY KEY (factor_id, symbol, date)
) PARTITION BY RANGE (date);

-- 按年分区
CREATE TABLE feature_values_2014 PARTITION OF feature_values
    FOR VALUES FROM ('2014-01-01') TO ('2015-01-01');
CREATE TABLE feature_values_2015 PARTITION OF feature_values
    FOR VALUES FROM ('2015-01-01') TO ('2016-01-01');
-- ... 每年一个分区

-- 按 date 的查询索引
CREATE INDEX idx_feature_values_date_factor ON feature_values(date, factor_id);
```

**性能考虑**：
- 200 因子 × 3000 股票 × 2500 交易日 ≈ 15 亿行（仅日 K）
- 分区后单分区约 1.5 亿行
- 查询模式：`(factor_id, symbol, date BETWEEN ... AND ...)`
- 预期查询延迟 < 50ms（主键直查）

## 5. 迁移表（需要改造）

```sql

-- Step 4: 新增其他字段

ALTER TABLE news ADD COLUMN IF NOT EXISTS related_industries JSONB;
ALTER TABLE news ADD COLUMN IF NOT EXISTS tags JSONB;
ALTER TABLE news ADD COLUMN IF NOT EXISTS url TEXT;

    ON news USING GIN (to_tsvector('simple', title || ' ' || coalesce(content,'')));
```

## 6. 索引策略

### 6.1 写密集型 vs 读密集型

| 表 | 写 | 读 | 策略 |
|----|---|---|------|
| `klines_all` | 每日批量 | 高频 | 最小化索引数，批量导入前先 drop index |
| `feature_values` | 夜间批量 | 高频 | 分区 + 主键 + 1 个辅助索引 |
| `session_events` | 实时高频 | 中频 | 主键 + 按 session 查询索引 |
| `inference_logs` | 实时高频 | 低频 | 只做日期分区，基本无索引 |

### 6.2 索引清单（核心）

```sql
-- 读热点索引
CREATE INDEX idx_klines_symbol_tf_dt_desc ON klines_all(symbol, timeframe, dt DESC);
CREATE INDEX idx_news_published_at_desc ON news(published_at DESC);
CREATE INDEX idx_feature_values_date_factor ON feature_values(date, factor_id);

-- 外键索引
CREATE INDEX idx_sessions_user ON sessions(user_id, created_at DESC);
CREATE INDEX idx_orders_session ON orders(session_id, ts);

-- 条件索引（部分）
CREATE INDEX idx_models_training ON models(status) WHERE status = 'training';
CREATE INDEX idx_sessions_active ON sessions(status)
    WHERE status IN ('initializing', 'running', 'paused');

-- JSONB GIN 索引
CREATE INDEX idx_news_related_symbols ON news USING GIN (related_symbols);
CREATE INDEX idx_models_metadata ON models USING GIN (metadata);
```

### 6.3 反索引禁忌

**不要**做的：
- `klines_all` 上加 `(open, high, low, close)` 组合索引（无查询场景）
- `session_events` 的 `payload` 字段整体 GIN 索引（太大）
- 小表上加多个索引（<1000 行没必要）

## 7. RLS 权限策略

所有用户数据表都开启 RLS：

```sql
ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;
ALTER TABLE training_samples ENABLE ROW LEVEL SECURITY;
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE models ENABLE ROW LEVEL SECURITY;
ALTER TABLE factor_definitions ENABLE ROW LEVEL SECURITY;
-- ...
```

### 7.1 典型策略

```sql
-- 1) 用户的私有资源只能自己访问
CREATE POLICY "sessions_owner" ON sessions
    FOR ALL USING (user_id = auth.uid());

-- 2) 公开资源任何人可读
CREATE POLICY "models_public_read" ON models
    FOR SELECT USING (visibility = 'public');

-- 3) 官方模型只有 service_role 能写
CREATE POLICY "models_platform_write" ON models
    FOR INSERT WITH CHECK (
        owner != 'platform' OR (SELECT current_setting('role') = 'service_role')
    );

-- 4) 匿名可读公开数据（行情/新闻）
CREATE POLICY "klines_anon_read" ON klines_all
    FOR SELECT TO anon USING (true);
CREATE POLICY "news_anon_read" ON news
    FOR SELECT TO anon USING (true);

-- 5) 训练样本只能本人 + 已贡献的样本（去标识）
CREATE POLICY "samples_owner" ON training_samples
    FOR ALL USING (user_id = auth.uid());
```

### 7.2 service_role 绕过 RLS

后端 Python 服务使用 service_role key 调用，绕过所有 RLS。**永远不能把 service_role key 暴露给客户端**。

## 8. Redis 缓存设计

### 8.1 Key 命名规范

```
<namespace>:<resource>:<id>[:<sub>]
```

示例：
```
factor:stats:ma_cross_20_60                 # 因子统计缓存
feature:600519:rsi_14:latest                # 最新因子值
model:abc123:meta                           # 模型元数据
session:sess_xxx:state                      # 会话实时状态
session:sess_xxx:events:queue               # 会话事件流缓冲
infer:signal:model_abc:600519:2024-03-15    # 推理结果
ratelimit:user_xxx:factors:1m               # 速率限制计数
pubsub:factor_updated                       # Pub/Sub 频道
```

### 8.2 数据结构选型

| Key 模式 | Redis 类型 | TTL | 用途 |
|---------|----------|-----|------|
| `factor:stats:*` | String (JSON) | 1h | 因子统计缓存 |
| `feature:*:latest` | Hash | 1d | 最新因子值快查 |
| `model:*:meta` | Hash | 5m | 模型元数据 |
| `session:*:state` | Hash | 无（会话结束清除） | 会话状态 |
| `session:*:events:queue` | List (LPUSH/RPOP) | 2h | 事件缓冲 |
| `infer:signal:*` | String (JSON) | 2h | 推理结果 |
| `ratelimit:*` | String (INCR + EXPIRE) | 按窗口 | 速率限制 |
| `HOT_MODELS` | Set | 无 | 常驻内存的模型列表 |

### 8.3 Pub/Sub 频道

| 频道 | 发布者 | 订阅者 | 用途 |
|------|-------|-------|------|
| `factor:updated` | feature-svc | infer-svc | 因子值更新通知 |
| `model:status` | train-svc | 前端 (via Orchestrator) | 模型状态变化 |
| `session:*:tick` | session-orch | 所有参与者 (WS) | 时钟推进 |

### 8.4 容量预估

- 单个会话事件缓冲：~100 条 × 2KB = 200KB
- 并发活跃会话：100 ≈ 20MB
- 因子值缓存：200 因子 × 3000 股票 × 100B = 60MB
- 推理结果缓存：1000 组合 × 1KB = 1MB
- **合计约 100MB**，1GB Redis 实例足够。

## 10. 数据生命周期

### 10.1 采集 → 存储 → 归档

```
BaoStock/AKShare
    ↓ 定时任务
feature-svc 采集
    ↓
klines_all 主表（热数据）
    ↓ 6 个月后（仅 5m）
自动迁移到 S3 Parquet
    ↓
主表保留日 K，分钟级查归档
```

### 10.2 因子值计算流

```
每日 17:00 收盘
    ↓
Celery Beat 触发
    ↓
feature-svc 增量计算当天因子值
    ↓
写入 feature_values（带分区）
    ↓
Redis Pub/Sub "factor_updated"
    ↓
infer-svc 刷新缓存
```

### 10.3 会话数据流

```
会话创建 → sessions 行
    ↓
期间事件 → session_events (高频写)
事件也进 Redis 缓冲（用于 WebSocket 广播）
    ↓
会话结束 → summary 计算 → 写入 sessions.summary
    ↓
若 data_collection_opt_in=TRUE
    ↓
生成 training_samples (等 N 日后再回填 label)
```

## 迁移 NAS 的规划（触发条件：买了 NAS）

### Phase 1 · NAS 部署（1 天）
- 装 Docker + Tailscale
- 启动 `supabase/docker-compose.yml` stack
- 开 HTTPS（Caddy/Traefik）

### Phase 2 · 数据迁移（半天）
- `pg_dump` 从 Supabase Cloud 导出全部 schema + data
- `pg_restore` 到 NAS PostgreSQL
- 校验行数一致

### Phase 3 · 切换（5 分钟）
```bash
# 只改一处：~/.investdojo.env
SUPABASE_URL=https://sb.nas.yourdomain.com   # 原：adqznqsciqtepzimcvsg.supabase.co
```

### Phase 4 · 补数据（1 天）
停写期间漏的数据（日 K / 市场快照）：
```bash
# 补跑 update_daily_klines.py 的日期区间参数
python scripts/update_daily_klines.py --from 2026-05-01 --to <NAS切换日>
```

### Phase 5 · 归档裁剪（按需）
NAS 磁盘自由度高，但仍建议：
- `klines_all` 保留 2 年，更早的 parquet 化扔 MinIO
- `feature_values` 保留 365 天，更早的 parquet

详细脚本见 `scripts/archive_to_parquet.py`（待写）。

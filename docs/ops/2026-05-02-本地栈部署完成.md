# 本地 Supabase 精简栈部署完成

> **日期**：2026-05-02
> **状态**：Done · Windows 端栈就绪，等 Mac 端导入数据
> **关联**：替代 [2026-05-01-停写说明.md](./2026-05-01-停写说明.md) 的 NAS 计划

---

## 📌 TL;DR

Supabase Cloud 免费档 500MB 磁盘满了之后，我们选择**在 Windows 电脑上自托管**替代品，**代码零改动**。

不是"裸 Postgres"，也不是"Supabase 全栈"，是**精简三件套**：
- **postgres** （数据）
- **postgrest** （代码里 `SupabaseClient` 打这个）
- **gotrue** （前端 `@supabase/ssr` 打这个）
- **kong** （把两者合到同一个 URL，和云端路径一致）

总内存占用 ~600MB，代码所有 `SupabaseClient` / `@supabase/supabase-js` 调用都能直接跑通。

---

## 🎯 为什么这么选（决策记录）

讨论过 4 个方案：

| 方案 | 代码改动 | 工作量 | 最终评价 |
|---|---|---|---|
| A · 完整 Supabase 全栈（10+ 容器） | 0 | 2 小时部署 + 12GB 内存 | 过重，没用上的组件一大堆 |
| B · 精简三件套（postgrest + gotrue + postgres） | 0 | 2 小时部署 | **选这个** ⭐ |
| C+ · 裸 PG + 改代码直连 | 改 5 个服务 + 3 个前端文件 | 4-5 天 | Epic 3 前不划算 |
| D · 换 MySQL / SQLite | 改 42 张表 DDL + 全部代码 | 2 周 | 灾难 |

**关键洞察**：审计后发现前端已深度绑定 Supabase Auth（`@supabase/ssr` + 5 个页面），C+ 的"零改动"前提不成立。方案 B 用官方 GoTrue 容器保留 Auth 能力，代码才能真正零改动。

**"分两段走"策略**：现在用方案 B 先上线（代码零改动），等 Epic 7 前再决定是否演进到 C+（彻底去 Supabase）。两段之间的升级路径是平滑的——方案 B 的 PG 数据和任何方案完全兼容。

## 🐛 踩过的坑（修复有据，别再踩）

| 坑 | 原因 | 修复位置 |
|---|---|---|
| `GOTRUE_SMTP_PORT` 空字符串让 Go 转 int 炸 | `${SMTP_PORT:-}` 在 compose 里展开成空 | `docker-compose.yml` 给合法占位 `:2500` |
| `API_EXTERNAL_URL missing` | 我忘了在 auth 容器注入 | `docker-compose.yml` 加 `API_EXTERNAL_URL` |
| Kong 用 `acl` 插件但没启用 | kong 3.x 默认不带 acl | `config/kong.yml` 干脆去掉 ACL，权限靠 JWT role claim |
| `must be owner of function uid` | init SQL 抢在 GoTrue 之前建了 `auth.uid()` | 让 `supabase_auth_admin` 拥有 `auth` schema，init 不碰函数 |
| `scenarios does not exist` | migrations/001 假设 scenarios 已存在（云端确实如此） | 新增 `init/01_legacy_tables.sql` 补 3 张老表基础结构 |
| PowerShell `Get-Content` 把 UTF-8 SQL 读成 GBK | SQL 里有中文注释 | `apply-migrations.ps1` 改用 `docker cp` 避开 PS 编码层 |
| PS 5.1 不支持 `??` 运算符 | up.ps1 最初版用了 | 改成 `if-else` |
| PS 5.1 把 SQL 乱码 | up.ps1 最初版写了中文 | 脚本改成 ASCII-only，中文全走 README |

## 🔄 和原 NAS 计划的差异

原 [2026-05-01-停写说明.md](./2026-05-01-停写说明.md) 的 NAS 规划作废（见运维手册 §8 备选方案），改用本方案。差异：

| 维度 | 原 NAS 计划 | 本方案（Windows 精简栈） |
|---|---|---|
| 硬件 | 需要买 NAS（~2000 元） | 用现有 Windows 电脑 |
| 部署时间 | 1-2 天 | 2-3 小时（含踩坑） |
| 容器数 | Supabase 全栈 10+ | 4（精简版） |
| 域名 + HTTPS | 要 | 不要（局域网 HTTP） |
| Mac 端访问 | Tailscale 或公网 | mDNS `investdojo.local` |
| 代码改动 | 0 | 0 |

Windows 电脑不 24/7 开机的话可以加个 Wake-on-LAN，或者 Mac 开发时临时开机即可（因为整个 Epic 3-8 开发本身也需要开机跑 Python 服务和前端）。

## 第一部分：环境搭建

### 1.1 必装工具清单

在 Terminal（终端）中依次执行以下命令。打开方式：`⌘ + 空格` 搜索 "Terminal"。

#### ① Homebrew（Mac 的包管理器）

```bash
# 如果你已经有 Homebrew，跳过这步
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

验证：`brew --version` 应该输出版本号。

#### ② Node.js 20+（JavaScript 运行时）

```bash
# 推荐用 fnm（Fast Node Manager），方便切换版本
brew install fnm

# 在 ~/.zshrc 中加入初始化代码（只需一次）
echo 'eval "$(fnm env --use-on-cd --shell zsh)"' >> ~/.zshrc
source ~/.zshrc

# 安装 Node.js 22（项目要求 >=20）
fnm install 22
fnm use 22
fnm default 22
```

验证：
```bash
node --version    # 应该输出 v22.x.x
npm --version     # 应该输出 10.x.x
```

#### ③ pnpm 9+（更快的包管理器，替代 npm）

```bash
# 开启 corepack（Node.js 自带的包管理器管理工具）
corepack enable

# 安装项目指定版本
corepack prepare pnpm@9.15.0 --activate
```

验证：`pnpm --version` 应该输出 `9.15.0`。

> **为什么用 pnpm 不用 npm？**  
> pnpm 安装速度快 2-3 倍，磁盘占用少 50%，而且天然支持 monorepo（一个仓库多个子项目）。

#### ④ Git（版本控制）

```bash
# Mac 通常自带 Git，检查一下
git --version

# 如果没有，安装
brew install git

# 基础配置（换成你的信息）
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

#### ⑤ 编辑器：VS Code

```bash
brew install --cask visual-studio-code
```

安装后，打开 VS Code，按 `⌘ + Shift + P`，输入 `shell command`，选择 **Install 'code' command in PATH**。这样你就可以在终端用 `code .` 打开项目了。

## 第二部分：30 分钟 React + TypeScript 速查

你不需要系统学完 React 和 TypeScript 才能开始写代码。以下是你在这个项目中 **真正会用到的** 语法子集。

### 2.1 TypeScript 核心语法（10 分钟）

TypeScript = JavaScript + 类型标注。你写的还是 JS，只是多了类型检查帮你提前发现 bug。

```typescript
// ===== 1. 变量类型 =====
const name: string = "贵州茅台";     // 字符串
const price: number = 1688.5;        // 数字
const isUp: boolean = true;          // 布尔值
const tags: string[] = ["白酒", "龙头"];  // 数组

// ===== 2. 对象类型（interface）=====
// 定义一个"形状"，描述对象长什么样
interface Stock {
  symbol: string;       // 必填
  name: string;
  price: number;
  change?: number;      // ? 表示可选
}

// 使用
const stock: Stock = {
  symbol: "600519",
  name: "贵州茅台",
  price: 1688.5,
  // change 没写也不报错，因为是可选的
};

// ===== 3. 函数类型 =====
// 参数和返回值都标注类型
function calcProfit(buyPrice: number, sellPrice: number, quantity: number): number {
  return (sellPrice - buyPrice) * quantity;
}

// 箭头函数（更常用）
const calcProfit2 = (buyPrice: number, sellPrice: number, quantity: number): number => {
  return (sellPrice - buyPrice) * quantity;
};

// ===== 4. 联合类型 =====
type Direction = "buy" | "sell";   // 只能是这两个值之一
type Status = "pending" | "done" | "error";

// ===== 5. 泛型（看到 <T> 不用怕，就是"类型参数"）=====
// 你暂时不需要自己写泛型，能看懂就行
// 比如 useState<number>(0) 表示"这个状态的值是 number 类型"

// ===== 6. 导入/导出 =====
// 从其他文件导入
import { Stock } from "./types";                // 具名导入
import type { Stock } from "./types";           // 仅导入类型（推荐）
import { SimulationEngine } from "@investdojo/core";  // 从包导入

// 导出给其他文件用
export interface Stock { /* ... */ }            // 具名导出
export default function MyComponent() { /* ... */ }  // 默认导出
```

### 2.2 React 核心概念（15 分钟）

React 的核心思想：**UI = f(state)**。你描述"数据是什么样时 UI 长什么样"，React 帮你自动更新。

#### 组件 = 函数

```tsx
// 一个 React 组件就是一个返回 JSX 的函数
// JSX 看起来像 HTML，但实际是 JavaScript

function StockCard({ stock }: { stock: Stock }) {
  // ↑ 参数叫 "props"（属性），用解构 {} 取出
  
  return (
    <div className="p-4 rounded border">
      {/* className 就是 HTML 的 class，因为 class 是 JS 保留字 */}
      <h2>{stock.name}</h2>
      {/* {} 花括号里可以写任何 JS 表达式 */}
      <p style={{ color: stock.change && stock.change > 0 ? "red" : "green" }}>
        ¥{stock.price.toFixed(2)}
      </p>
    </div>
  );
}

// 使用组件：像 HTML 标签一样
<StockCard stock={{ symbol: "600519", name: "贵州茅台", price: 1688.5 }} />
```

#### 状态管理（useState）

```tsx
"use client";  // ← Next.js 规则：用了 useState 等交互功能的组件要加这行

import { useState } from "react";

function Counter() {
  // useState 返回 [当前值, 修改函数]
  const [count, setCount] = useState(0);
  //     ↑ 读         ↑ 写         ↑ 初始值

  return (
    <div>
      <p>点击了 {count} 次</p>
      <button onClick={() => setCount(count + 1)}>
        点我 +1
      </button>
    </div>
  );
}
```

#### 副作用（useEffect）

```tsx
"use client";
import { useState, useEffect } from "react";

function StockPrice({ symbol }: { symbol: string }) {
  const [price, setPrice] = useState<number | null>(null);

  // useEffect = "组件渲染后要做的事"
  useEffect(() => {
    // 这里可以发请求、设定时器等
    fetch(`/api/stock/${symbol}`)
      .then(res => res.json())
      .then(data => setPrice(data.price));
    
    // 返回清理函数（组件销毁时执行）
    return () => {
      // 清理定时器等
    };
  }, [symbol]); // ← 依赖数组：symbol 变化时重新执行

  if (price === null) return <p>加载中...</p>;
  return <p>当前价格：¥{price}</p>;
}
```

#### 条件渲染 & 列表渲染

```tsx
function StockList({ stocks }: { stocks: Stock[] }) {
  return (
    <div>
      {/* 条件渲染：&& 短路 */}
      {stocks.length === 0 && <p>暂无数据</p>}
      
      {/* 条件渲染：三元表达式 */}
      {stocks.length > 0 ? (
        <ul>
          {/* 列表渲染：.map() + key */}
          {stocks.map((stock) => (
            <li key={stock.symbol}>{stock.name}: ¥{stock.price}</li>
          ))}
        </ul>
      ) : (
        <p>没有数据</p>
      )}
    </div>
  );
}
```

### 2.3 本项目用到的状态管理：Zustand

Zustand（德语"状态"）比 Redux 简单 10 倍。项目用它管理全局状态。

```tsx
// 定义 store（只需写一次）
import { create } from "zustand";

interface BearStore {
  bears: number;
  increase: () => void;
  decrease: () => void;
}

const useBearStore = create<BearStore>((set) => ({
  bears: 0,
  increase: () => set((state) => ({ bears: state.bears + 1 })),
  decrease: () => set((state) => ({ bears: state.bears - 1 })),
}));

// 在任何组件中使用（自动响应变化）
function BearCounter() {
  const bears = useBearStore((state) => state.bears);
  const increase = useBearStore((state) => state.increase);
  return <button onClick={increase}>🐻 {bears}</button>;
}
```

### 2.4 Tailwind CSS（样式速查）

本项目用 Tailwind CSS 写样式 — 不用写 CSS 文件，直接在 className 里写工具类。

```tsx
// 传统 CSS 写法：
// .card { padding: 16px; border-radius: 8px; background: white; }
// <div className="card">...</div>

// Tailwind 写法（等价）：
<div className="p-4 rounded-lg bg-white">...</div>
```

常用类速查：

| 类名 | 效果 | CSS 等价 |
|------|------|---------|
| `p-4` | 内边距 16px | `padding: 1rem` |
| `m-2` | 外边距 8px | `margin: 0.5rem` |
| `px-4` | 左右内边距 | `padding-left/right: 1rem` |
| `mt-2` | 上外边距 | `margin-top: 0.5rem` |
| `flex` | 弹性布局 | `display: flex` |
| `gap-4` | 间距 16px | `gap: 1rem` |
| `grid grid-cols-3` | 三列网格 | `display: grid; grid-template-columns: repeat(3, 1fr)` |
| `text-sm` | 小字 14px | `font-size: 0.875rem` |
| `text-red-500` | 红色文字 | `color: #ef4444` |
| `bg-gray-900` | 深灰背景 | `background: #111827` |
| `rounded-lg` | 圆角 8px | `border-radius: 0.5rem` |
| `border` | 1px 边框 | `border: 1px solid` |
| `hover:bg-gray-800` | 悬停变色 | `&:hover { background: ... }` |
| `md:grid-cols-3` | 中屏以上三列 | `@media (min-width: 768px) { ... }` |
| `hidden md:block` | 小屏隐藏，中屏显示 | 响应式 |

> 💡 不用背，[Tailwind 文档](https://tailwindcss.com/docs) 支持搜索，比如搜 "padding" 就能找到所有内边距的类。

### 3.2 包依赖关系（谁依赖谁）

```
@investdojo/web (网页应用)
  ├── @investdojo/core  (业务逻辑)
  ├── @investdojo/ui    (UI 组件) ── → @investdojo/core
  └── @investdojo/api   (API 客户端) ── → @investdojo/core

@investdojo/server (后端 API)
  └── @investdojo/core  (业务逻辑)
```

**核心原则**：`core` 是基础层，所有人都依赖它，它不依赖任何人。

### 3.3 "我想改 XX 功能，该改哪个文件？"

| 我想改... | 去哪里 | 文件 |
|----------|-------|------|
| 交易规则（手续费、涨跌停） | `packages/core` | `simulation/engine.ts` |
| K 线图表的颜色 / 样式 | `packages/ui` | `charts/KLineChart.tsx` |
| 交易面板的按钮和布局 | `packages/ui` | `trading/OrderPanel.tsx` |
| 模拟交易页面的整体布局 | `apps/web` | `simulation/[scenarioId]/SimulationPlayground.tsx` |
| 场景列表页面 | `apps/web` | `simulation/ScenarioListPage.tsx` |
| Mock 历史数据 | `apps/web` | `lib/mock-data.ts` |
| 后端 API 接口 | `apps/server` | `routes/simulation.ts` 或 `routes/ai.ts` |
| 首页 | `apps/web` | `app/page.tsx` |

## 第五部分：日常开发工作流

### 5.1 修改代码 → 实时预览

项目使用 **HMR（热模块替换）**，保存文件后浏览器 **自动刷新**，无需手动操作。

```
修改代码 → Ctrl+S 保存 → 浏览器自动更新（<1秒）
```

### 5.2 典型开发流程

```
1. pnpm dev:web                    ← 启动服务
2. 打开浏览器 localhost:3000       ← 预览
3. 在 VS Code 中修改代码           ← 开发
4. Ctrl+S 保存 → 浏览器自动刷新   ← 实时预览
5. 重复 3-4                        ← 迭代
6. pnpm type-check                 ← 最后检查类型错误
7. git add . && git commit         ← 提交
```

### 5.3 常用开发命令

```bash
# ===== 日常开发 =====
pnpm dev:web              # 启动网页开发服务器
pnpm dev:server           # 启动后端开发服务器
pnpm dev                  # 同时启动所有

# ===== 类型检查（发现错误）=====
pnpm type-check           # 检查所有包的类型错误（重要！）

# ===== 构建（部署前）=====
pnpm build                # 构建所有包
pnpm build:web            # 只构建网页

# ===== 清理（遇到奇怪问题时）=====
pnpm clean                # 清理构建产物
rm -rf node_modules && pnpm install   # 核弹级清理：删掉所有依赖重装

# ===== 添加新依赖 =====
# 给某个子包添加依赖
pnpm --filter @investdojo/web add axios        # 给 web 加 axios
pnpm --filter @investdojo/core add dayjs       # 给 core 加 dayjs
pnpm --filter @investdojo/ui add -D @types/xxx # 给 ui 加开发依赖
```

### 5.6 修改 core 逻辑

修改 `packages/core` 中的代码后，由于 core 需要编译，有两种方式让前端拿到最新代码：

```bash
# 方式 1：core 开启 watch 模式（推荐，自动编译）
```

```tsx
function OrderPanel({ stock }) {
```

### 6.3 TypeScript 类型错误

类型错误会在两个地方显示：

1. **VS Code 中**：红色波浪线 + 鼠标悬停看错误信息
2. **终端中**：运行 `pnpm type-check` 查看所有类型错误

常见类型错误和解决方法：

```typescript
// 错误 1: Property 'xxx' does not exist on type 'YYY'
// 原因: 访问了不存在的属性
// 解决: 检查拼写，或在 interface 中添加该属性

// 错误 2: Type 'string' is not assignable to type 'number'
// 原因: 类型不匹配
// 解决: 检查变量的实际类型，做类型转换

// 错误 3: 'xxx' is possibly 'undefined'
// 原因: 值可能不存在
// 解决: 加 ?. 可选链或 if 判断
const name = stock?.name;              // 可选链
const name = stock ? stock.name : "";  // 条件判断
const name = stock!.name;              // 断言（你确定它存在时用）

// 错误 4: Cannot find module '@investdojo/core'
// 原因: core 包没有构建
// 解决: pnpm --filter @investdojo/core build
```

### 7.3 单元测试（核心引擎）

为 `packages/core` 写测试是最优先的，因为撮合引擎的逻辑必须正确。

```typescript
// packages/core/src/simulation/__tests__/engine.test.ts

import { describe, it, expect } from "vitest";
import { SimulationEngine } from "../engine";

describe("SimulationEngine 撮合引擎", () => {
  
  it("应该正确执行买入交易", () => {
    const engine = new SimulationEngine();
    const result = engine.executeBuy({
      symbol: "600519",
      price: 1688.5,
      quantity: 100,
      cash: 1_000_000,
    });
    
    // 买入费用 = 价格 × 数量 + 佣金
    expect(result.success).toBe(true);
    expect(result.filledQuantity).toBe(100);
    expect(result.totalCost).toBeGreaterThan(168850);  // 含佣金
  });

  it("资金不足时应该拒绝交易", () => {
    const engine = new SimulationEngine();
    const result = engine.executeBuy({
      symbol: "600519",
      price: 1688.5,
      quantity: 100,
      cash: 100,  // 只有 100 块
    });
    
    expect(result.success).toBe(false);
    expect(result.error).toContain("资金不足");
  });

  it("涨停价应该无法买入", () => {
    const engine = new SimulationEngine();
    const result = engine.checkBuyable({
      symbol: "600519",
      currentPrice: 1857.35,  // 涨停价
      prevClose: 1688.5,
    });
    
    expect(result.buyable).toBe(false);
  });

  it("应该正确计算收益率", () => {
    const profit = SimulationEngine.calcReturn(100, 110);  // 买100卖110
    expect(profit).toBeCloseTo(0.1);  // 10%
  });
});
```

在 `packages/core/package.json` 的 scripts 中添加：

```json
{
  "scripts": {
    "test": "vitest",
    "test:run": "vitest run"
  }
}
```

运行测试：

```bash
# 交互式（文件变化自动重跑）
pnpm --filter @investdojo/core test

# 跑一次（CI 用）
pnpm --filter @investdojo/core test:run
```

### 7.4 组件测试（UI）

```tsx
// packages/ui/src/trading/__tests__/OrderPanel.test.tsx

import { describe, it, expect } from "vitest";
import { render, screen, fireEvent } from "@testing-library/react";
import { OrderPanel } from "../OrderPanel";

describe("OrderPanel 交易面板", () => {
  const defaultProps = {
    currentPrice: 100,
    cash: 1_000_000,
    onBuy: vi.fn(),
    onSell: vi.fn(),
  };

  it("应该渲染买入/卖出按钮", () => {
    render(<OrderPanel {...defaultProps} />);
    expect(screen.getByText("买入")).toBeDefined();
    expect(screen.getByText("卖出")).toBeDefined();
  });

  it("点击买入应该触发回调", () => {
    render(<OrderPanel {...defaultProps} />);
    fireEvent.click(screen.getByText("买入"));
    expect(defaultProps.onBuy).toHaveBeenCalled();
  });
});
```

## 第八部分：常见问题排查

### 问题 1：`pnpm install` 失败

```bash
# 症状: ERR_PNPM_OUTDATED_LOCKFILE 或类似错误

# 解决: 删掉 lockfile 重装
rm pnpm-lock.yaml
pnpm install

# 如果还是不行，核弹清理
rm -rf node_modules apps/*/node_modules packages/*/node_modules pnpm-lock.yaml
pnpm install
```

### 问题 2：`pnpm dev:web` 启动报错 "Module not found"

```bash
# 症状: Cannot find module '@investdojo/core'

# 原因: core 包没有构建

# 解决: 先构建 core
pnpm --filter @investdojo/core build

# 或者全部构建
pnpm build
```

### 问题 3：页面空白 / 白屏

```bash
# 1. 打开浏览器开发者工具（F12），切到 Console 面板看报错
# 2. 常见原因：
#    - "use client" 忘了加（用了 useState/useEffect 的组件）
#    - import 路径拼写错误
#    - 缺少依赖：pnpm install

# 3. 终端看是否有编译错误（红色文字）
```

### 问题 4：样式不生效

```bash
# 症状: className 写了但没有效果

# 检查 1: postcss.config.cjs 是否存在
cat apps/web/postcss.config.cjs
# 应该包含 @tailwindcss/postcss

# 检查 2: globals.css 是否导入了 tailwind
cat apps/web/src/app/globals.css
# 应该有 @import "tailwindcss";

# 检查 3: layout.tsx 是否导入了 globals.css
# import "./globals.css" 应该在 layout.tsx 顶部
```

### 问题 5：端口被占用

```bash
# 症状: Error: listen EADDRINUSE: address already in use :::3000

# 找到占用端口的进程
lsof -i :3000

# 杀掉它
kill -9 <PID>

# 或者换个端口
pnpm --filter @investdojo/web exec -- next dev --port 3002
```

### 问题 6：Git 提交前检查

```bash
# 提交前建议跑一遍类型检查
pnpm type-check

# 如果有错误，根据提示修复

# 然后提交
git add .
git commit -m "feat(simulation): 添加涨跌停检测逻辑"
```

### 问题 7：Node.js 版本不对

```bash
# 症状: "engines" 版本不匹配

# 检查当前版本
node -v

# 切换版本
fnm use 22

# 如果没装 22
fnm install 22
fnm use 22
```

### 问题 8：TypeScript 到处报红但能运行

```bash
# 可能是 VS Code 的 TypeScript 服务没有更新

# 解决: 在 VS Code 中按 ⌘ + Shift + P
# 输入 "TypeScript: Restart TS Server"
# 回车重启
```
